# 股票研究数据库 - 产品规格文档

---

## 一、产品定位

本地部署的股票研究管理工具，用于整合选股、AI研究报告录入、分类管理、Watchlist导出的完整workflow。

**使用场景**：盘前/盘后，Momentum交易者使用。

**工作流程**：

1. 用券商选股器筛选股票
2. 在claude.ai中输入ticker，运行预设prompt，获得JSON + Markdown报告
3. 在本App中录入数据，管理分类和标签
4. 筛选导出Watchlist到TradingView

---

## 二、功能需求

### 模块1：数据录入

- 输入股票代码（Ticker）
- 粘贴AI生成的JSON字段（解析后存入数据库对应字段）
- 粘贴AI生成的Markdown报告（存为报告文件）
- 已存在的Ticker提示"更新 or 跳过"
- 每次保存自动记录生成时间戳

### 模块2：Table View（主界面）

- 显示所有股票的摘要字段
- 支持列的显示/隐藏（用户自定义）
- 支持按任意字段排序
- 支持多条件Filter筛选
- 导出Watchlist → TradingView格式（每行一个Ticker的纯文本）
- 支持多条件Filter筛选，新建watchlist
- 点击某行 → 右侧滑出详细报告Panel
- 软删除（移入Archived，可恢复）

### 模块3：详细报告 Panel

- 从右侧滑出，带关闭按钮，不离开Table页面
- 显示AI生成的Markdown报告内容
- 显示历史版本列表（同一股票多次更新的报告对比）
- ~~手动私人笔记区域（与AI报告分开存放）~~
- ~~显示生成时间、Tags、Category、状态~~

### 模块4：Watchlist View

- 左侧：TradingView Chart Widget（嵌入）
- 右侧：可折叠的Watchlist分组列表
- 点击Watchlist中某个Ticker，左侧Chart自动切换
- 支持多个Watchlist分组切换
- 导出Watchlist → TradingView格式（每行一个Ticker的纯文本）

### 模块5：设置（Settings）

- **Themes管理**：预设统一**Theme**列表，支持增删改；AI生成的**Theme**从列表中选取，也可自由添加
- **Category管理**：行业分类列表，支持增删改
- **Watchlist管理**：创建/重命名/删除Watchlist，管理Watchlist内的股票成员
- **数据库字段管理**：配置JSON扩展字段中哪些key显示为Table的列

## 三、技术方案

### 技术栈

| 层级 | 技术 | 说明 |
| --- | --- | --- |
| 前端框架 | React + TypeScript | 组件化，类型安全 |
| 前端构建 | Vite | 快速开发/构建 |
| 样式 | Tailwind CSS | 工具类样式 |
| UI组件库 | shadcn/ui | 轻量可定制，Table/Sheet/Dialog直接满足需求 |
| 后端 | Node.js + Express + TypeScript | 前后端统一语言 |
| 数据库 | SQLite + Prisma | Prisma管理schema，省去手写SQL |
| 版本管理 | Git + GitHub | 代码备份 + 历史记录 |

### 架构

```
本地 Web App
├── frontend/               # React + TypeScript + Vite
│   ├── src/
│   │   ├── components/     # shadcn/ui 组件
│   │   ├── pages/          # 各页面
│   │   └── lib/            # API调用封装
├── backend/                # Node.js + Express + TypeScript
│   ├── src/
│   │   ├── routes/         # API路由
│   │   └── prisma/         # 数据库schema
│   ├── prisma/
│   │   └── schema.prisma   # 数据库模型定义
│   └── reports/            # Markdown报告文件
│       └── NVDA_20250315.md
└── .gitignore
```

### 本地运行方式

```bash
# 后端
cd backend && npm run dev    # http://localhost:3000

# 前端
cd frontend && npm run dev   # http://localhost:5173
```

### TradingView Chart

- 使用 TradingView 免费 Widget（嵌入式，无需API key）
- 非实时数据，适合盘前/盘后使用

### GitHub同步

- 手动 `git push` 或设置定时备份脚本
- 报告Markdown文件和数据库都纳入版本管理

## 四、数据库设计

### 表1：stocks（股票主表）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER PK | 自增主键 |
| ticker | TEXT UNIQUE | 股票代码，如 NVDA |
| theme_id | INTEGER FK | 关联Theme表 |
| category_id | INTEGER FK | 关联Category表 |
| created_at | DATETIME | 首次录入时间 |
| updated_at | DATETIME | 最近更新时间 |
| is_archived | BOOLEAN | 软删除标记 |
| risk | TEXT | 最大风险 |
| catalyst | TEXT | 核心催化剂 |
| company_name | TEXT | 公司名称 |
| summary | TEXT | 一句话定位 |
| theme_benefit | TEXT | 直接受益 / 间接受益 / 概念蹭热度 |
| theme_phase | INTEGER | 早期 / 中期 / 末期 |

> `extra_data` 功能取消，不做扩展 JSON 字段存储与展示。
> 

---

### 表2：reports（报告版本表）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | INTEGER PK | 自增主键 |
| ticker | TEXT FK | 关联stocks表 |
| file_path | TEXT | Markdown文件路径 |
| generated_at | DATETIME | 本次生成时间 |
| version | INTEGER | 版本号，同一股票递增 |

> 每次更新报告生成新记录，旧版本保留，支持历史对比。
> 

---

### 表3：notes（私人笔记表）

> 功能取消，不再存储私人笔记。

---
